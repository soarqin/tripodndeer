# 04 · 军事系统 · 战争 · 将领

> 战争是大战略游戏最容易做成数字游戏的部分。我们要让它有"兵法"的味道。

---

## 1. 设计目标

| 目标 | 含义 |
|------|------|
| **战争是政治的延续** | 不是双方点击"开战"就完了；要有借口、过程、议和、影响 |
| **质数兼顾** | 兵力多寡重要，但兵种、地利、将领、士气同样关键 |
| **关隘是叙事舞台** | 函谷、虎牢这些地名要在玩家口中说出来 |
| **不操作微观** | 玩家下达战略指令，不直接操作单兵；让战斗有"决策密度"而非"操作密度" |

---

## 2. 军事系统的层级

```
战略 (Strategic)        ← 玩家关心：调几个军团去哪个方向
   │
战役 (Operational)      ← 玩家关心：是包围还是绕道，要不要在此地决战
   │
战术 (Tactical)         ← 主要由系统模拟：会战的损失、战法的发挥
```

玩家直接操作**战略层**和**部分战役层**。战术层基于双方将领、兵种、地形、士气**自动结算**，玩家可在战前观察预测、战中下达高级指令（如"撤退"、"奇袭"）。

## 3. 兵种系统（Unit Types）

### 3.1 基础兵种（春秋战国时期）

| 兵种 | 代表 | 强项 | 弱项 | 时代特征 |
|------|------|------|------|---------|
| **车兵** | 战车 | 平原冲击力 | 山地林泽几无用；造价昂贵 | 春秋主力，战国渐衰 |
| **步兵** | 徒卒 | 通用、便宜 | 平原对抗骑兵脆弱 | 全时代主力 |
| **弩兵** | 强弩 | 远程、克车骑 | 近战脆弱 | 战国成型，韩、魏强 |
| **骑兵** | 轻骑 | 机动、迂回、追击 | 攻城无用 | 战国后期成熟，赵改革后强势 |
| **重甲步兵** | 魏武卒 | 防御、阵地战 | 机动差、训练贵 | 魏国吴起改革产物 |
| **水军** | 楼船 | 江河运输、水战 | 仅水域可用 | 楚、吴、越发达 |
| **工兵** | 攻城营造 | 攻城、修工事 | 不参与野战 | 全时代 |

### 3.2 兵种的"剪刀石头布"

不是绝对克制，是修正：

```
弩  克  车 / 重步
车  克  普步 (平原)
骑  克  普步 / 弩 (开阔地)
重步  克  车 / 骑 (山地林泽)
水军   独立维度
```

平衡通过**地形**和**编制**来调节，避免一支兵种通吃。

### 3.3 时代差异

不同时代的兵种**完全不同**：
- 春秋战国：以上兵种
- 三国：增加马凯（铁骑）、楼船更成熟、井阑等攻城器械
- 五代十国：神臂弓、火药雏形（视乎历史考据）

> 通过剧本数据驱动，引擎不耦合。

## 4. 军队组织（Army Composition）

### 4.1 基本单位：军团（Army）

玩家不直接管理"一个士兵"，最小操作单位是**军团**。

```yaml
army:
  id: army_qin_west_01
  name: 秦西路军
  realm: realm_qin
  commander: char_bai_qi      # 主帅
  vice_commanders: [char_xxx, char_yyy]
  composition:
    infantry: 30000
    crossbow: 10000
    cavalry: 5000
    siege_engineers: 2000
  morale: 80
  experience: medium
  supply: 25                  # 粮草天数
  current_position: site_yongcheng
  state: marching             # marching | engaged | besieging | resting | retreating
  destination: site_handan
  orders: aggressive_march    # 玩家下达的指令
```

### 4.2 军团数量限制

每个势力可同时拥有**有限的军团**：
- 基础上限 = 3
- + 大将军任职：+1
- + 政体加成：变法后 +1
- + 军事改革（吴起改制、商鞅军功爵等）：+1~2

> 军团少而精——避免玩家乱点小队，迫使战略思考。

### 4.3 兵源与征兵

```yaml
manpower:
  pool: 50000           # 当前可用兵源
  cap: 80000            # 上限（由人口、忠诚、政体决定）
  monthly_recovery: 500 # 月度恢复（春耕慢、秋后快）
  conscript_max_per_op: 20000  # 单次最大征兵
```

**征兵约束**：
- 不能瞬间无中生有，需要时间（数月到数年）
- 大规模征兵会造成**民望下降**、**生产降低**
- 农忙时期征兵代价翻倍

### 4.4 军费与粮草

| 资源 | 影响 |
|------|------|
| **军饷** | 月度结算，欠饷会降低士气和忠诚 |
| **粮草** | 行军/驻扎日消，断粮则崩溃 |
| **战马** | 骑兵专用资源，需要北方/西方草场 |
| **兵器** | 弩、戈、矛、甲胄；可由内政生产储备 |

## 5. 战争前置：师出有名（Casus Belli）

### 5.1 为什么需要

- 历史现实：春秋讲礼，宣战需借口；战国渐崩，但仍有"罪状"
- 玩法收益：让宣战不是免费的，让外交博弈有意义
- 防止 AI 和玩家无脑乱打

### 5.2 战争借口类型

| 类型 | 触发条件 | 影响 |
|------|---------|------|
| **争位**（如周王册封他国为伯） | 受到周王或大国册封 | 合法 |
| **复仇**（祖辈被杀、领土被夺） | 历史事件 / 已宣示的仇敌 | 合法 |
| **救援盟友** | 盟约触发 | 合法、提升威望 |
| **救济伐暴** | 对方做出暴政（伐无道） | 合法 |
| **争夺继承** | 借嫡庶之争介入他国 | 半合法 |
| **无故入侵** | 没有借口 | 不合法（重大威望损失、合法性下降、外交孤立） |

### 5.3 借口的收益

- **合法宣战**：威望不损、停战条款可以更苛刻、占领地控制度高
- **不合法宣战**：威望大损、其他国家更可能围攻你、占领地控制度低（容易叛乱）

> 这是周天子作为势力的核心玩法之一——它可以**赋予**或**收回**他国的合法性。

## 6. 行军系统（March System）

### 6.1 行军的本质

军团沿邑邻接图移动。每条邻接边有**行军成本**（基础 = 该边 travel_cost × 编制平均速度修正 × 季节修正 × 地形修正）。

### 6.2 行军命令类型

| 命令 | 含义 | 速度 | 损耗 |
|------|------|------|------|
| **常规行军** | 标准阵型推进 | 1.0× | 1.0× |
| **强行军** | 加速但消耗大 | 1.4× | 1.5× |
| **急行军** | 极速，可能减员 | 1.8× | 2.0× |
| **掠夺行军** | 边走边抢，补给从敌地获取 | 0.8× | 0.5×（吃敌方补给） |
| **隐蔽行军** | 慢但敌方情报感知滞后 | 0.6× | 1.0× |

### 6.3 补给线

军团进入敌国后需要**补给线**：
- 自动从最近的我方城邑拉一条补给路径
- 补给路径上有敌军会被切断 → 军团饿着，士气狂掉
- 玩家可主动派军团**断敌粮道**

> 这让"长平之战"式的歼灭战在游戏中可以发生。

## 7. 战斗系统（Battle System）

### 7.1 三种战斗形态

| 形态 | 触发 | 特点 |
|------|------|------|
| **野战** | 两军在邑外接触 | 地形决定、机动决定 |
| **攻城战** | 军团围攻有城防的邑 | 长时间消耗、需攻城器 |
| **关隘战** | 军团强攻关隘 | 防御加成极高、需特殊战法 |

### 7.2 野战结算（自动 + 可观察）

战前**显示双方"沙盘"**：
- 双方兵力、兵种构成
- 双方主帅与副将
- 地形修正
- 士气与补给状态
- **预测胜率**（根据情报覆盖度，可能不准）

战斗持续若干 Tick（旬），**滚动结算**伤亡：
- 玩家可看进程
- 可中途下达指令（撤退、改打法）
- 可追加援军、断粮

战后输出：
- 双方伤亡
- 战利品 / 缴获
- 主帅生死（有概率）
- 士气变化
- 战役级别叙事文本（"赵将廉颇拒秦于长平，秦不能进……"）

### 7.3 攻城战

**关键参数**：
- 城防值（base + 工事 + 守军）
- 守军数量与补给
- 攻方攻城器械数
- 围城时间

**结局途径**：
1. **强攻**：消耗大但快（攻方损失高）
2. **困城**：等其断粮或瘟疫，慢但损失小
3. **里应外合**：通过谍报系统发动叛乱（详见 [`05-systems-statecraft.md`](./05-systems-statecraft.md)）
4. **议和退兵**：达成条件即撤
5. **守军投降**：困到士气崩溃

### 7.4 关隘战的特殊性

- 关隘默认 +50%~+80% 防御
- 攻关战法 / 工程兵效率决定推进速度
- **奇袭/反间**事件链可绕过常规战斗（高风险高回报）
- 大型关隘可能要打数月乃至数年

## 8. 将领系统（General Mechanics）

### 8.1 主帅的影响

- **武**：直接战斗力修正
- **谋**：识破对方计谋、计算战法
- **学**：阵型部署多样性
- **统御**：能指挥的兵力上限（避免一员将领统百万雄师）

### 8.2 战法（Tactics）

战法是将领可使用的**战斗特殊动作**。每个战法消耗"将略点"，每场战斗有限。

| 战法 | 学习路径 | 效果 |
|------|---------|------|
| **奇正之策** | 兵家学习 | 双方混战时倒转优势 |
| **声东击西** | 谋士辅助 | 敌方误判主攻方向 |
| **诱敌深入** | 高谋将领 | 引敌进入伏击点 |
| **背水一战** | 高武将领+士气低 | 残兵爆发战斗力 |
| **围师必阙** | 高谋将领 | 围城战提高敌方士气崩溃概率 |
| **以战养战** | 掠夺型 | 占敌邑后立即获得补给 |
| **疲敌之计** | 高谋 | 长期对峙削弱敌士气 |

> 战法的解锁与运用让懂兵法的玩家有"我会用孙子兵法"的爽感。

### 8.3 将领死亡

- 将领可在战斗中阵亡（概率与"亲临前线"等设置有关）
- 名将之死是**历史性时刻**，应自动暂停 + 庄重通知
- 玩家可选"重金抚恤"以保持军心

## 9. 战争结束（War Resolution）

### 9.1 议和（Peace Treaty）

任何一方可发起议和。条款选项：

| 条款 | 影响 |
|------|------|
| **割地** | 转移指定邑/郡的归属 |
| **赔款** | 一次性国库转移 |
| **岁币 / 朝贡** | 数年期分期付款 |
| **释放战俘** | 返还被俘将领 |
| **签订盟约** | 战后转为盟友（罕见） |
| **册封从属** | 战败方成为附庸 |
| **解散军团** | 战败方解散指定军团 |
| **划界禁建** | 边境若干年不得筑城 |
| **退还合法性** | 重大威望转移 |

议和由**双方接受意愿**计算：
- 战场局势
- 国力对比
- 第三国压力
- 双方战疲值

### 9.2 占领与吞并

- **占领**：战时控制对方邑，但归属未变
- **吞并**：议和条款转移归属，需要时间消化（控制度从低开始爬升）
- **直辖 vs 设附庸**：胜方可选择把战败方设为附庸而非直接吞并

### 9.3 战疲（War Exhaustion）

战时民望、生产、稳定性持续下降。**长期战争**会引发：
- 流民、起义
- 兵源枯竭
- 国库空虚
- 内部派系不稳

> 让"穷兵黩武"自然带来恶果，而非靠数值惩罚。

## 10. 异族战争（Barbarian Wars）

> 中国古代战争的另一面是"夷狄之患"。

### 10.1 异族势力

- 不参与中原"礼制"系统（无合法性、无朝贡、不讲师出有名）
- 倾向掠夺式战争（来去如风、不占地）
- 在某些时代是重要威胁（春秋楚被视为蛮夷、战国西戎、北狄）

### 10.2 长城/边墙系统

- 各国可在边境邑修建边墙（特殊建筑）
- 边墙降低异族入侵概率，但不能完全防御
- 历史上的长城前身正是这套系统的演化结果

## 11. 待确认问题（Open Questions）

| # | 问题 | 备选 |
|---|------|------|
| OQ-1 | 战斗动画细致度？ | 推荐：地图上动态的兵团图标 + 战斗弹窗内的分阶段叙事文本，不做单兵动画 |
| OQ-2 | 是否引入海战 / 水战的独立系统？ | 推荐：首发不做独立水战，水军仅作为运兵和封锁用 |
| OQ-3 | 围城战是否做"长平之战"式的特别长战役？ | 推荐：是。用事件链+战时叙事让这种战役有标志性 |
| OQ-4 | 攻城器械是否需要单独建造与运输？ | 推荐：是。运输受地形限制（增加战略深度） |
| OQ-5 | 是否允许玩家"亲征"？ | 推荐：是。亲征加成大但风险高（可能被俘、薨于军中） |
| OQ-6 | 兵种是否随科技/改革升级？ | 推荐：是。改革是解锁新兵种的主要途径（赵→骑射、魏→武卒） |
