# 02 · 地图系统 · 领土涂色 · 地形 · 关隘

> 地图是主角，本作的灵魂。本文档定义地图的结构、视觉、归属规则和战略要素。

---

## 1. 设计目标

| 目标 | 含义 |
|------|------|
| **地理可信** | 玩家一眼能认出"这是中原"、"这是楚地"、"这是燕北" |
| **战略有意义** | 山川河流不是装饰，是兵家必争之地的成因 |
| **涂色有重量** | 一抹色块的扩张/收缩，能让玩家感受到文明气运 |
| **信息分层清晰** | 远观是版图，近看是城邑；缩放即信息层级切换 |
| **可扩展** | 同一地图引擎应能承载春秋战国 / 三国 / 五代十国等多个时代 |

---

## 2. 空间结构（Spatial Structure）

### 2.1 三级空间层次

```
天下 (World)
  └── 州/部 (Region)        ← 文化与地理分区，约 9–12 个
        └── 郡 (Province)   ← 中等行政单元，约 80–150 个/时代
              └── 邑 (Site) ← 最小归属单元，约 400–800 个/时代
```

| 层级 | 数量级 | 角色 |
|------|------|------|
| **邑（Site）** | 数百 | 领土归属的最小单元；可被驻军、围攻、建设、涂色 |
| **郡（Province）** | 百级 | 行政管理单元；税赋、人口、文化在此层结算；通常含一座主城 + 若干属邑 |
| **州（Region）** | 一二十 | 地理与文化分区（如"关中"、"中原"、"江南"、"巴蜀"、"燕代"）；用于宏观策略和事件触发 |

> **数据驱动**：每个时代的剧本通过 JSON 重新定义所有邑/郡/州，引擎不变。详见 [`06-scenarios.md`](./06-scenarios.md)。

### 2.2 邑（Site）的属性

```yaml
# 示例：邯郸（赵国都城）
id: site_handan
name: 邯郸
type: capital_city                    # 类型：都城 / 主城 / 属邑 / 边塞 / 关隘 / 港埠
position: [x, y]                      # 地图坐标
terrain: plains                       # 地形
adjacency: [shexian, gaoyi, wuxiang]  # 相邻邑 ID 列表（决定行军图）
defense_value: 80                     # 城防值（防御工事强度）
population_base: 50000                # 基础人口
economy: { agri: 5, craft: 3, trade: 4 }  # 经济属性
cultural: chinese_central             # 文化标签
historical_owner: zhao                # 历史归属（用于剧本/事件）
```

### 2.3 郡的属性
- 由若干邑组成
- 含一座**主城邑**（行政中心）
- 拥有郡级数值（人口总量、税收、忠诚度、文化认同）
- 可被任命**太守**（详见 [`03-factions.md`](./03-factions.md)）

### 2.4 州的属性
- 由若干郡组成
- 主要用于：宏观地图缩放显示、AI 战略层、事件触发条件、文化归属
- 不参与详细模拟

## 3. 邑的邻接关系（Adjacency Graph）

### 3.1 为什么用图，不用网格

我们**不**使用六边形或方格网格。原因：
- 网格游戏感太重，破坏地图的史诗感
- 真实地理（山川阻隔、河流走向）无法用规则网格表达
- 想要"函谷关易守难攻"的效果，必须打破规则邻接

**采用「邑—邑邻接图」**：每个邑明确声明它和哪些邑相邻，行军、贸易、外交都沿这张图传播。

### 3.2 邻接边的属性

每条邻接边携带额外信息：

```yaml
edge: { from: handan, to: shexian }
type: open                       # open | mountain_pass | river_crossing | sea_route
travel_cost: 1.0                 # 行军时间倍率
strategic_pass: null             # 若为关隘，指向关隘 ID（影响攻防）
```

### 3.3 关隘是邻接边的"门"

详见后文「关隘系统」。简言之：**关隘不是一个邑，而是某条邻接边上的关闸**。打开了邻接，走到关隘门口；想要继续前进，必须破关。

## 4. 领土归属与涂色（Ownership & Painting）

### 4.1 归属模型

每个邑有**唯一**的归属势力（owner_realm_id）。但归属不只是 0/1 的开关，还有**强度**（control level）：

```yaml
site_state:
  owner: zhao                    # 当前归属势力
  control_level: 100             # 控制度 (0–100)
  contested: false               # 是否处于争夺中
  occupier: null                 # 若被敌军占领，记录占领方
  loyalty: 85                    # 民心忠诚度
  cultural_identity: chinese_zhao # 文化认同（可与归属不一致）
```

### 4.2 控制度（Control）的来源

控制度 = max(0, 基础值 + 修正值)
- **基础值** 100（刚拿下的领土在过渡期内逐步建立）
- 修正值受以下影响：
  - **驻军**（+）
  - **太守能力**（+）
  - **文化认同匹配**（+ 或 −）
  - **被叛军占领**（−）
  - **敌军兵临城下但未克**（−）
  - **瘟疫/灾害**（−）

### 4.3 涂色规则（Painting Rules）

| 视觉状态 | 含义 | 视觉表现 |
|---------|------|---------|
| **稳固归属** | control_level ≥ 80 | 实色 + 势力主色 |
| **薄弱控制** | 50 ≤ control_level < 80 | 半透明该色 + 斜线纹理 |
| **争夺中** | contested = true | 红色边框闪烁 + 双方颜色拉锯条纹 |
| **被占** | occupier ≠ owner | 占领方颜色 + 原归属虚线边框（提示是"占而未取"） |
| **空地/无主** | owner = null | 浅灰色，不参与势力涂色 |
| **叛乱区** | rebellion_active | 棕色 / 黑色斑驳 + "🔥" 图标 |

### 4.4 涂色过渡动画

当一个邑的归属变更时，颜色变化**不是瞬间**的：
- 0.3 秒 ease-in 渐变到新颜色
- 同时播放**轻量音效**（克城为低沉鼓声、收复为编钟声）
- 镜头**轻微**抖动或飞向新涂色区域（仅在玩家持有该势力时）

> 这是"地图的史诗"支柱的具体落地——让一次涂色具有"仪式感"。

### 4.5 文化认同与归属的分离

一个邑可能：
- 政治归属于秦
- 文化认同还是赵

这是**叛乱/复国/同化**系统的根源。详见 [`05-systems-statecraft.md`](./05-systems-statecraft.md) 的文化章节。

## 5. 地形系统（Terrain）

### 5.1 地形类别

| 地形 | 视觉 | 行军成本 | 防御加成 | 经济基础 |
|------|------|--------|---------|---------|
| **平原** | 浅黄绿 | 1.0 | +0% | 高农业 |
| **丘陵** | 棕黄 | 1.3 | +20% | 中农业 |
| **山地** | 深棕带峰纹 | 2.0 | +50% | 低农业，可矿 |
| **林木** | 深绿 | 1.4 | +25% | 中农业，可猎 |
| **沼泽** | 暗灰绿斑点 | 1.8 | +30% | 极低农业 |
| **大河** | 蓝带 | — (依靠桥渡口) | 桥渡战防+40% | 渔业、运输 |
| **海岸** | 蓝灰 | — | 港埠类型 | 渔业、海贸 |
| **草原** | 米黄 | 0.9 (骑兵 0.7) | -10% (开阔) | 畜牧 |
| **荒漠** | 沙黄 | 1.6 | -20% | 极低 |

### 5.2 地形的双重作用

1. **数值修正**：影响行军、防御、产出
2. **视觉叙事**：让玩家一眼看出"这片是富饶的关中平原"、"这里是难以攻克的巴蜀险地"

### 5.3 季节-地形交互

- 冬季的大河可结冰（可踏冰直渡，绕开关隘）
- 雨季的沼泽不可通行
- 夏季的山林易爆瘟疫

→ 详见 [`01-core-loop.md`](./01-core-loop.md) 季节系统。

## 6. 关隘系统（Strategic Passes）

> 这是本作最具中国古代特色的系统之一。

### 6.1 为什么需要关隘

中国古代战争史的核心叙事之一就是**关隘**：
- 函谷关让六国伐秦数次受阻
- 虎牢关使关东诸侯不能西进
- 剑阁让蜀地"一夫当关、万夫莫开"

如果不专门建模关隘，地图就只是一张普通的军事地图，失去文明的厚度。

### 6.2 关隘的本质

**关隘不是一个邑，而是"邻接边上的关闸"**：

```
邑A ──[关隘]── 邑B
```

- 想要从邑A 行军到邑B，**必须**经过关隘
- 关隘有自己的城防值、驻军、归属
- 攻方不破关，则不能进入对面邑（除非有第三条迂回路径）

### 6.3 关隘的属性

```yaml
pass:
  id: pass_hanguguan
  name: 函谷关
  controls_edges: [(luoyang, xian)]  # 控制哪条/哪些邻接边
  owner: qin                         # 归属（通常跟随邻接邑之一）
  garrison: 5000                     # 驻军
  fortification: 90                  # 关防强度
  terrain_bonus: 60                  # 地利加成（山势险要）
  flank_difficulty: very_hard        # 迂回难度（影响是否可绕过）
  historical_significance: critical  # 历史重要性（影响事件触发）
```

### 6.4 关隘的玩法影响

1. **攻方必须先破关**才能继续推进
2. 关隘的防御加成 = 关防强度 + 地利加成（远高于普通城防）
3. 关隘可以被**迂回**，但代价巨大（绕远路、损耗大、可能遭遇野战）
4. 关隘可以被**强攻**，但需要专门攻坚部队和长时间围困
5. 关隘可以被**奇袭**（间谍/叛降），但触发需要特定事件链

### 6.5 关隘列表（首发剧本预设）

> 春秋战国首发地图建议包含这些关隘（不完全列举）：

| 关隘 | 控制 | 战略意义 |
|------|------|---------|
| 函谷关 | 关中⇄中原 | 秦守此关则进可攻退可守 |
| 武关 | 关中⇄南阳 | 秦灭楚的南路 |
| 萧关 | 关中⇄陇西 | 防义渠戎 |
| 大散关 | 关中⇄汉中 | 通巴蜀 |
| 太行八陉（部分）| 中原⇄河北 | 多关节点 |
| 虎牢关 | 中原⇄洛阳 | 京畿门户 |
| 武胜关 | 中原⇄江汉 | 楚地北门 |
| 居庸关 / 古北口 | 燕地⇄北方 | 防胡 |

具体是否全部建模，看首发地图工作量；至少**函谷关、武关、虎牢关**必须有。

## 7. 渲染层级与缩放（Rendering Levels）

### 7.1 三级缩放

| 缩放级别 | 显示主体 | UI 元素 |
|---------|---------|--------|
| **战略层 / 远观** | 州着色、势力主色块 | 国名、君主、势力盘整态势 |
| **战役层 / 中观** | 郡着色、主要城邑、行军线 | 部队动向、关隘高亮、贸易线 |
| **战术层 / 近观** | 每个邑、地形纹理、单位 | 详细驻军、建筑、人物头像 |

> 类似 EU4 / CK3，但**不分模式**——同一个地图、连续缩放，不要做"mapmode 切换"那种打断式分屏。

### 7.2 地图模式（Mapmodes）

但**信息覆盖层**可切换：

| 模式 | 用途 |
|------|------|
| 政治模式（默认） | 势力涂色 |
| 文化模式 | 文化分布涂色 |
| 经济模式 | 富庶程度热力图 |
| 军事模式 | 驻军强度热力图 |
| 外交模式 | 玩家与其他势力的关系（盟友/敌对/中立） |
| 地形模式 | 仅显示地形 |
| 谍报模式 | 显示情报覆盖度 |

切换是图层叠加，**不**重新加载地图。

## 8. 视觉风格（Visual Style）

### 8.1 美学方向

**水墨工笔结合**：
- 远观水墨写意（远景山川用墨色泼染）
- 近看工笔细描（城池、关隘、文字考究）
- 配色克制，不浓艳

**纸本质感**：
- 整张地图有古旧纸张底纹
- 边缘略有泛黄
- 绘制风格似古地图（可参考《禹贡九州图》《华夷图》）

### 8.2 配色系统

势力主色应选**辨识度高且文化吻合**的色彩。例：

| 势力 | 主色 | 文化考据 |
|------|------|---------|
| 周（王室） | 朱赤 | 周尚赤 |
| 秦 | 玄黑 | 秦尚黑（水德） |
| 齐 | 青蓝 | 东方木德 / 海色 |
| 楚 | 暗赤 | 楚尚赤，南方火德 |
| 晋（早期） | 苍白 | 中原文气 |
| 韩 | 橘黄 | — |
| 赵 | 紫青 | — |
| 魏 | 翠绿 | — |
| 燕 | 灰白 | 北方水冷 |

> 设计时考虑色盲友好：所有相邻势力色差应通过明度+色相双维度区分。

### 8.3 字体

- 中文：使用**思源宋体**或类似的开源衬线宋体（古朴庄重）
- 关键大字（国名、城名）可用**衬线楷体**
- 数字与西文：搭配**思源等宽**或类似

详见 [`09-ux-ui.md`](./09-ux-ui.md)。

## 9. 性能注意事项

### 9.1 渲染性能

- 邑级别量级（数百），可全部用 SVG/Canvas 实时渲染
- 但**地形纹理**应预渲染为位图，避免实时绘制
- 涂色变更应是**局部 redraw**，不是整张重绘
- 缩放使用 transform，不重新生成几何

### 9.2 内存占用

- 邑邻接图作为常驻内存数据（结构小）
- 地形位图按需加载/卸载（首次缩放时按区块加载）
- 历史动画帧不做完整快照（仅事件级快照）

详见 [`10-tech.md`](./10-tech.md)。
