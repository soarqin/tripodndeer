> 决定本作"摸起来是什么手感"的最关键文档。

## 1. 核心循环（The Loop）

```
        ┌─────────────────────────────────────────────────────────┐
        │                                                         │
        ▼                                                         │
   [观察地图]                                                     │
        │                                                         │
        ▼                                                         │
   [接收事件 / 通知 / 警报]                                       │
        │                                                         │
        ▼                                                         │
   [玩家选择是否暂停]   ←──── 重大事件可触发自动暂停             │
        │                                                         │
        ▼                                                         │
   [在暂停态中:                                                   │
     · 浏览详情                                                   │
     · 调整军令                                                   │
     · 颁布政令                                                   │
     · 外交斡旋                                                   │
     · 任免人才]                                                  │
        │                                                         │
        ▼                                                         │
   [恢复时间，1x / 2x / 3x / 4x / 5x 速度推进]                   │
        │                                                         │
        ▼                                                         │
   [模拟引擎步进 (按 Tick 推进)]                                  │
        │                                                         │
        └─────────────────────────────────────────────────────────┘
```

**关键观察**：循环的"决策密度"完全由玩家控制。
- 玩家可以"快速观光"——挂 5 倍速看局势演变
- 也可以"沉浸经营"——遇事就停，每个细节都琢磨
- 这不是 bug，这是 feature：**节奏由玩家选择**

## 2. 时间制（Time System）

### 2.1 时间单位

| 单位 | 时长 | 在游戏中的角色 |
|------|------|---------------|
| **Tick / 旬** | 10 天 | 模拟引擎的最小推进单位；行军、生产、谍报的最小结算粒度 |
| **月** | 3 旬 | 月度结算（税收、粮草、政令进度） |
| **季** | 3 月 | 季节系统（影响行军、产出、瘟疫） |
| **年** | 4 季 | 年度结算（人口、人才老化、君主寿命） |

> **为什么是「旬」而不是「日」**
> 1. 中国古代历法本就以"上中下旬"分月，文化贴合
> 2. 1 天为 Tick 会导致模拟过密、性能压力大、玩家信息疲劳
> 3. 1 月为 Tick 又过粗，行军指挥失去精细度
> 4. 10 天是行军、谍报、消息传递的合理粒度（古代信使日行 200 里左右）

### 2.2 时间速度

| 档位 | 实时秒/Tick | 推进速度 | 适用场景 |
|------|-----------|---------|---------|
| ⏸ 暂停 | ∞ | — | 决策、阅读、规划 |
| ▶ 1x | 5.0 秒 | 慢 | 战争中、紧张博弈 |
| ▶▶ 2x | 2.5 秒 | 较慢 | 日常推进 |
| ▶▶▶ 3x | 1.5 秒 | 中速 | 默认速度 |
| ▶▶▶▶ 4x | 0.8 秒 | 快 | 长途行军、内政积累期 |
| ▶▶▶▶▶ 5x | 0.4 秒 | 最快 | 跳过和平期 |

> 数值可在 [`10-tech.md`](./10-tech.md) 中调优。
> 关键原则：**5x 模式下也不应卡顿**。模拟和渲染需异步、增量更新。

### 2.3 季节系统（Seasons）

| 季节 | 月份（农历近似） | 玩法影响 |
|------|---------------|---------|
| **春** | 1–3 月 | 春耕开始；适宜出征（粮草接续）；瘟疫低概率 |
| **夏** | 4–6 月 | 农忙；不宜大规模征兵（伤生产）；河水盛，不利涉渡 |
| **秋** | 7–9 月 | 秋收；粮草丰沛；古典"秋高马肥"，骑兵+10% |
| **冬** | 10–12 月 | 农闲，可大规模征发；北方苦寒，行军损耗+20%；河流可踏冰直渡 |

> 季节不只是数值修正，更是节奏感来源：玩家会自发形成"秋后用兵"的古典战略直觉。

## 3. 暂停哲学（The Philosophy of the Pause）

### 3.1 暂停不是"停止游戏"，而是"切换到决策模式"

我们把游戏时间分为两个**逻辑模式**：

| 模式 | 游戏在做什么 | 玩家在做什么 |
|------|-----------|-------------|
| **运行态（Running）** | 模拟世界推进，事件持续触发 | 观察、感受、判断局势 |
| **决策态（Paused）** | 时间冻结，但所有 UI 仍可交互 | 思考、查询、规划、下达命令 |

**关键设计**：暂停态下玩家几乎可以做任何事——查看势力详情、下达军令、外交沟通、颁布政令——**唯一不能做的就是看时间走**。

这是 Paradox 风格的核心：**Pause is for thinking, not for stopping.**

### 3.2 自动暂停（Auto-Pause）

某些事件**默认触发自动暂停**，但玩家可在设置中关闭每一项：

| 触发事件 | 默认是否自动暂停 | 理由 |
|---------|---------------|------|
| 君主薨逝 / 继承危机 | ✅ | 重大不可逆决策（继承人选择） |
| 宣战通知（无论攻防） | ✅ | 战争状态切换是质变 |
| 重大历史事件触发（如三家分晋通过） | ✅ | 剧情节点不应被加速跳过 |
| 谍报：刺杀任务结果 | ✅ | 高风险事件结果 |
| 玩家任命的关键人才死亡 | ✅ | 人才系统的关键节点 |
| 重要会战结束 | ✅ | 让玩家看到战果 |
| 边境异族入侵 | ✅ | 突发危机 |
| 普通战斗胜负 | ❌ | 太频繁，会打断节奏 |
| 月度税收结算 | ❌ | 日常事件 |
| 普通外交回复 | ❌ | 日常事件 |

### 3.3 玩家自定义暂停

玩家可在**通知设置**中为每类事件单独配置：
- 是否在通知栏出现
- 是否触发自动暂停
- 是否在小地图上闪烁高亮

→ 这让"信息噪音"问题完全交给玩家自己解决。

### 3.4 不可逆决策的折返窗口

对于**真正不可逆**的决策（如颁布"焚书"政令、册立太子、主动宣战），UI 必须：
1. 弹出确认窗口
2. 列出**预期后果**（合法性变化、势力关系变化、长期 Buff/Debuff）
3. 提供"取消"按钮
4. **执行前**保留 1 个自动存档（用于反悔）

> 反悔不是惩罚玩家，是保护沉浸体验。

## 4. 玩家行为分类（Player Action Taxonomy）

按"决策频率"和"影响周期"对玩家行为做分层，便于 UI 与提示系统设计：

### 4.1 频繁、即时、可逆（Tactical · 战术）

- 调整军队行军路线
- 修改防守驻军
- 调整城市生产队列
- 切换贸易路线

**特征**：每旬可能多次发生，影响范围小，错了能改。
**UI 要求**：拖拽、快捷键、最少点击。

### 4.2 偶发、需思考、半可逆（Operational · 战役）

- 委派人才到某城担任太守
- 与某国签订盟约
- 派遣使者求和
- 调动主力军团出征
- 设置间谍任务

**特征**：以月或季为单位发生，影响有时滞，撤回有代价。
**UI 要求**：聚焦窗口、显示后果预览、二次确认。

### 4.3 罕见、关键、近不可逆（Strategic · 战略）

- 改革政治制度（如商鞅变法）
- 迁都
- 颁布意识形态法令（独尊儒术等）
- 主动宣战大国
- 称王 / 称帝

**特征**：以年甚至十年为单位发生，影响整局走向。
**UI 要求**：庄重的确认流程、详尽的后果说明、自动存档。

## 5. 信息流设计（Information Flow）

### 5.1 信息分级

| 级别 | 触发动作 | 视觉强度 | 例子 |
|------|---------|---------|------|
| **L1 · 警报** | 自动暂停 + 居中弹窗 + 音效 | 最强 | 君主薨逝、首都被围 |
| **L2 · 提醒** | 顶部横幅 + 通知栏闪烁 | 强 | 邻国宣战、重大灾祸 |
| **L3 · 通知** | 通知栏静默累积 | 中 | 普通会战结束、外交回复 |
| **L4 · 日志** | 仅记录于事件日志，不打扰 | 弱 | 月度结算、贸易完成 |

### 5.2 通知聚合（Notification Aggregation）

为防止信息过载，同类事件会**聚合**为单条通知：
- "边境 3 处发生小规模冲突"（而非 3 条独立通知）
- "本旬完成 5 项城市建设"（而非 5 条）

但**关键事件**（L1/L2）永不聚合，必须独立呈现。

### 5.3 历史日志（Chronicle）

游戏全程记录一份**编年体史书**——每一场战役、每一次结盟、每一次君主更替，都以古典史笔的口吻记入：

> *秦昭襄王二十二年，冬。秦遣白起率十万众伐韩，克野王。韩王使张平如秦请和，秦不许。*

这份日志：
- 玩家可随时翻阅
- 胜利后可导出为"史记"分享
- 失败后亦可导出为"亡国录"
- 服务支柱一（历史的重量）+ 体验目标（玩家会和别人讲故事）

## 6. 决策密度与节奏曲线

理想的**单局玩家决策密度曲线**：

```
决策密度
  ▲
  │      ╱╲                     ╱╲
  │     ╱  ╲       ╱╲          ╱  ╲
  │    ╱    ╲     ╱  ╲        ╱    ╲
  │   ╱      ╲___╱    ╲______╱      ╲
  │__╱                                ╲__
  └────────────────────────────────────────▶
   开局   外交   小战   危机   决战    一统
```

每个"波峰"对应一个**节点性事件**：
- 开局了解势力，部署人才
- 第一次外交博弈
- 第一次军事冲突
- 一次中期危机（继承、内乱、强敌联军）
- 决战阶段的连续大决策
- 终局的政治整合

设计时**主动制造**这些波峰（通过事件链、AI 行为、历史触发器），而不是依赖随机事件堆砌。

## 7. "无聊"的设计权利（The Right to Be Calm）

并非每一旬都要有事发生。允许（甚至鼓励）出现"和平积累期"：
- 玩家挂 4–5 倍速跳过 2 年
- 看着自己的国家慢慢变强
- 等待下一个事件触发

**这是 Paradox 风格的精髓**：高潮与平静交替，让玩家有"我在统治一个国家"的真实感，而不是"我在不停被事件追打"。

> 不要为了避免"无聊"而往游戏里塞事件。让玩家有静下来欣赏地图的时刻。
