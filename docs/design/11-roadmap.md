# 11 · MVP 路线图与里程碑

> 设计文档不能落地，就是空中楼阁。本文给出**实现路径**：先做什么、后做什么、什么时候算"可玩"。

---

## 1. 设计哲学

### 1.1 核心循环优先

> 在任何子系统精雕细琢之前，必须先让**核心循环**跑通。

错误做法：先做精美军事系统 → 再发现没有外交根本无法成局
正确做法：先让最简单的"地图 + 时间 + 涂色 + 一键征战"跑通，看手感对不对，再深化

### 1.2 垂直切片优先（Vertical Slice）

每个里程碑都应交付**一个能玩的版本**，而非"完成 30% 的所有系统"。

### 1.3 内容与代码分离开发

数据驱动让**美术 / 内容 / 历史考据**可与编程并行。

### 2.1 当前节奏与依赖说明（重要）

M4 v1 已交付最小可行经济（参见 §7）。但 M4 设计文档（[`05-systems-statecraft.md`](./05-systems-statecraft.md) §3）中的"变法"、"灾害"、"派系"等内容**依赖人才系统**，因此**实施顺序重排**：

```
M4 (✅) ──► M5 (✅) ──► M4.1 (✅) ──► M4.2 (✅) ──► M6 (✅) ──► M7 (✅) ──► M8 (✅) ──► M9 (✅) ──► Wave 9 (✅)
```

**为什么 M5 提前**：变法需要"革新者"人才（商鞅、吴起），文化扩散需要"学者"产人才，谍报需要"间者"人才——所以人才骨架是后续所有里程碑的**共同前置**。把 M5 提到 M4.1 之前做，避免重复重构。

**编号语义不变**：M5/M6/M7 依然分别指代"人物 / 文化 / 谍报"，仅**实施顺序**调整。M4.1/M4.2 是 M4 主题的收尾补充，做在 M5 之后。

## 3. M0 · 引擎雏形（最最最小可行）

### 3.1 目标

**让一个红色块和一个蓝色块在地图上随时间演变。**

不是开玩笑——这是让所有人确认"哦，这个游戏的'手感'是这样"的关键。

### 3.2 必须完成

- ✅ Vite + React + TypeScript 项目初始化
- ✅ Canvas 地图渲染（载入硬编码 JSON 的 5 个邑）
- ✅ 时钟系统（暂停 + 5 档速度）
- ✅ 涂色系统（每隔 N 个 Tick 红色随机吞噬一个蓝色块）
- ✅ 顶栏（仅时间显示）
- ✅ 时间控制条
- ✅ Engine / Rendering / UI 三层分明的目录结构

### 3.3 不需要

- ❌ 真实的春秋战国地图
- ❌ 任何 UI 美术
- ❌ AI（用纯随机替代）
- ❌ 存档
- ❌ 任何子系统

### 3.4 验收标准

- 能加载、暂停、加速
- 看着 5 个色块的归属在变化
- 帧率稳定 60 FPS
- 主创团队达成共识："是的，这个手感就是我们要的"

> **这一步千万不要跳过**。它可能 2 周就能做完，但能避免后面 6 个月白做。

## 4. M1 · 核心循环可玩

### 4.1 目标

**玩家可以扮演秦国，宣战韩国，派军占领韩的一座城，让它变成秦的颜色。**

### 4.2 必须完成

- 引入正经的春秋战国地图（简化版：仅七雄+周，每势力 5–10 个邑）
- 引入"势力"实体（国号、颜色、起始邑）
- 引入"军团"实体（兵力数、所在邑、行军中状态）
- 玩家行动：
  - 选择军团
  - 右键邑 → "进军" / "宣战并进军"
  - 行军到达 → 占领（涂色变化）
- AI（最简）：
  - 其他六国按 80% 概率不动、20% 概率随机攻击邻国
- 主面板雏形：王宫面板（仅显示自己势力总览）
- 底栏雏形：仅"军事"按钮（其他禁用）

### 4.3 不需要

- 兵种、地形修正、关隘、将领
- 完整外交、内政
- 历史人物
- 事件链
- 存档

### 4.4 验收标准

- 玩家能玩 30 分钟一局，最终把地图涂红
- 不卡顿、不报错
- 主创团队能说"这就是中国版的 EU4 雏形"

## 5. M2 · 战争系统

### 5.1 目标

**让"战争"这件事有兵法味道。**

### 5.2 必须完成

- 兵种（步、车、骑、弩，4 种入门）
- 地形修正
- 关隘系统（首批 5 个核心关隘）
- 攻城战（区分野战 vs 围城）
- 将领（武力、统御、忠诚）
- 战法（5–8 种入门）
- 师出有名（基础 5 种战争借口）
- 议和（割地、赔款、岁币）
- 战疲与兵源池

### 5.3 验收标准

- 玩家会感叹"原来这个游戏不是数兵打架"
- 关隘在玩家口中说出来（"我守住函谷关就赢了"）
- AI 会做基本战术（围攻、断粮、撤退）

## 6. M3 · 外交系统 ✅ 已交付

### 6.1 目标

**让玩家和 AI 不是只能打仗，还能谈、能盟、能背叛。**

### 6.2 已交付内容（v1）

- 双边关系（态度 -100~+100、信任 0~100，对等存储于 `World.relations`）
- 外交动作 7 种：盟约、互不侵犯、朝贡（仅元数据，经济效果在 M4 接入）、联姻、使节、宣战、议和
- 提案/条约生命周期（pending → accepted/rejected/expired/cancelled；条约 active/expired/cancelled/broken）
- 第三国感知反应（盟友被攻打 → 关系档自动调整）
- 简易合纵（M3 阶段：根据霸主威胁的反向施压）
- 周王册封状态（最简：`ZhouInvestitureState`，影响外交评分）
- AI 自然形成"群雄抗秦"格局（合纵参与决策）
- M3.1 完整 UI 中文化（DiplomacyPanel / PeacePanel / TopBar / TimeControlBar）

### 6.3 验收（已通过）

- AI vs AI 自动跑分中，弱小相邻势力围攻强势霸主的情形可观察
- 玩家在 DiplomacyPanel 看到 7 种动作并可提交，背叛会取消相关条约
- M3 e2e 通过（参见 `.sisyphus/evidence/task-8-m3-diplomacy-e2e.txt`）

### 6.4 M3 没做（明确推迟）

- 完整朝贡 / 册封事件链 → 推到 M6（文化与意识形态）
- 间者 / 暗杀 / 离间 → 推到 M7
- 外交人才（辩士、纵横家）的实际加成 → 推到 M5

## 7. M4 · 内政与经济（v1） ✅ 已交付

### 7.1 目标

**让和平期有事可做：粮、钱、税以月度结算的形式让玩家在不打仗时也有决策。**

### 7.2 已交付内容（v1）

- **资源体系（最小可行）**：势力级 `treasury`（铜钱）、`foodStores`（粮）、`taxRate`（0–50% 整数）；邑级 `population`、`households`（= 人口/5）、`taxBase`、`foodProduction`
- **月度结算 phase**：纯函数 `economyPhase(world, rng)`，仅在 `date.xun === 'shang'` 触发；执行顺序排在 `diplomacyLifecycle` 之后
- **政令系统（v1）**：2 种实装政令——`edict_tax_relief`（税收 −25% / 人口增长 +）、`edict_grain_reserve`（粮产 + / 月耗钱）；含 `durationMonths` 倒计时与 `expired` 状态
- **太守系统（v1）**：邑级任命，固定修饰 2 种——`tax_efficiency`、`food_efficiency`；将领 `学/谋/stat` 不影响产出（M5 之前不做）
- **M3 议和经济效果接入**：`indemnity`（一次性国库转账）、`tribute`（按月转账，clamp 到 ≥ 0）实际生效
- **UI 经济面板**：M4 Economy 面板显示国库/粮/税率/月度增量/邑列表/激活政令/任命太守
- **平衡数值集中**：所有 M4 数值常量以 `M4_` 前缀放在 `src/content/m2/balance.ts`

### 7.3 验收（已通过）

- 100 tick p95 < 性能预算（参见 `.sisyphus/evidence/task-10-perf.json`）
- 同种子 + 同初始世界 → 经济结算 byte-equal 可复现
- M0/M1/M2/M3 加载路径无回归（生成 50 个 site 不变）
- e2e：玩家可激活政令、推进时间、看到月度变化、任命太守

### 7.4 M4 v1 明确**未做**（按依赖延后）

| 延后内容 | 推到何处 | 原因 |
|---|---|---|
| 变法（商鞅 / 吴起 / 胡服骑射） | **M4.1** | 依赖 M5 革新者人才 |
| 灾害与丰歉（旱涝瘟蝗） | **M4.2** | 依赖 M5 太守属性影响赈灾效果 |
| 贸易商路 | **M4.2** | 与外交/文化耦合，需先有 M6 文化标签 |
| 内部派系（公族/世族/军功派/改革派/守旧派/外姓客卿） | **M4.2** | 依赖 M5 人才任命 |
| `popularMorale` / `prestige` / `legitimacy` | **M5** | 这些维度的来源是君主与人才 |
| 完整 20+ 政令种类 | **M4.1 / M4.2** 滚动加 | M4 v1 仅 2 种作为系统骨架 |
| 募兵 UI | **M5** | 依赖人口×太守一致性 |

## 8. M5 · 人物与人才 🔄 下一里程碑

> M5 是 **后续所有里程碑的依赖根**——M4.1（变法）、M4.2（派系）、M6（学宫产人才）、M7（间者）都建立在 M5 的人才骨架之上。

### 8.1 目标

**让"人"出现在游戏里：君主有命数，人才有去向，任命有意义。**

### 8.2 必须完成

#### 8.2.1 君主（Ruler）骨架

- 君主属性 6 维实装：`武 / 政 / 交 / 谋 / 学 / 魄`（0–20 整数）
- 君主健康值 + 寿命（基础 60–70，受年龄/事件递减）
- 君主薨逝事件 → 自动暂停 → 继承流程
- 继承法 v1：仅嫡长子继承制；无合法储君时进入"继承危机"
- 继承危机 v1：3 种解决（强立旁系 / 兄终弟及 / 内乱→旧君势力分裂为 1–2 个新势力）
- 君主性格原型 8 种：Conqueror / Steward / Schemer / Learned / Tyrant / Incompetent / Benevolent / Builder（影响 AI 评分权重，详见 [`07-ai.md`](./07-ai.md) §3）
- `World.rulers: ReadonlyMap<RealmId, RulerState>` + Zod schema

#### 8.2.2 人才（Character）骨架

- 人才核心属性 6 维（同君主）+ `loyalty 0–100` + `ambition: low/mid/high`
- 人才专长 9 种：统帅 / 猛将 / 谋士 / 吏才 / 革新者 / 辩士 / 间者 / 学者 / 工师
- 人才来源 v1：① 历史出生（剧本数据驱动） + ② 野生人才（命名池随机）；学宫产出推到 M6
- 人才任命岗位 v1：相 / 大将军 / 太守（M4 已有但仅作为身份指针）；其他岗位（御史 / 太尉 / 使节）骨架字段保留，效果推后续
- 人才离心：忠诚低于阈值 → `怠工 / 求去 / 私通 / 叛变` 4 状态机

#### 8.2.3 与 M4 的接入

- 太守任命**开始查询**人才 `政` 属性影响 `tax_efficiency` 强度（取代 M4 v1 的固定常量）
- 大将军任命**开始查询**将领 `武 / 统御` 影响军团兵力上限（M2 之前固定）
- 相国任命影响**未来 M4.1 变法**触发条件

#### 8.2.4 历史人物事件链 v1（≥3 个）

- 蔺相如完璧归赵
- 范雎远交近攻
- 廉颇老矣（赵括代将）
- 苏秦 / 张仪：先实装"辩士"专长字段，事件链推到 M5 末尾或 M6

> **注**：M5 历史人物事件链定位为长平之战年代（公元前 260 年起）的真实将相事件。早于游戏起始年的商鞅/吴起已迁移到 M4.1 作为可选变法 trigger。

### 8.3 验收标准

- 玩家会说："我那一局商鞅没去秦国，结果魏国变法了"——意味着事件链有玩家可见的史实分歧
- 君主薨逝时游戏自动暂停，弹出继承窗口，玩家选项有合法性后果预览
- 任命人才到岗位后，相关数值变化在 UI 面板可见（不是黑箱）
- 所有 8 种性格原型至少各跑通一次 AI 决策测试（区别于 M3 的 3 种简易性格）

### 8.4 M5 不做（明确边界）

- 完整家族 / 血缘 / 联姻树（CK 风深度）→ 推到 v2
- 人物头像与立绘（仅占位）→ 推到 M10
- 历史人物特殊技能（管仲「相国之政」、白起「人屠」）→ 推到 M9（剧本内容包）
- 装备 / 经验值 / 技能树（养成手游路线，**永远不做**）
- 人物对话与情感系统 → 推到 v2
- 早于游戏起始年的人物事件链（如商鞅/吴起本人入仕）不在 M5 范围

### 8.5 关键决策点（实施前必须确认）

- **决策 D1**：君主性格 8 种是 M5 全量实装，还是先 4 种（Conqueror / Steward / Schemer / Learned）？
- **决策 D2**：派系系统（公族 / 世族 / 军功派...）骨架是否在 M5 引入，还是完全推到 M4.2？推荐：M5 引入**字段**（`character.faction`），效果推到 M4.2
- **决策 D3**：继承危机时"势力分裂"是否要在 M5 实装，还是仅文本提示？推荐：M5 实装最小版本（分 2 个新势力）

## 9. M4.1 · 变法系统 ✅ 已交付

> 在 M5 人才骨架完成后，M4.1 让变法系统真正发生在游戏里。

### 9.1 目标

**让单局有戏剧性的政治转折点：变法 = 不可逆的结构性改革。**

### 9.2 必须完成

- 变法事件链 v1：胡服骑射、申不害变法、稷下辩论、楚国吴起遗产
- 变法触发条件：① 在场革新者人才（M5 专长）+ ② 君主性格 ∈ {Builder, Conqueror} + ③ 政治稳定窗口
- 变法多年事件链：阻力 → 流血 → 成功/失败分支（每步玩家有选择）
- 成功效果：永久势力 trait（如 `shang_yang_reform_done` → 兵源 +20% / 内政效率 +30%），覆盖至少 5 个数值维度
- 失败反弹：人才被诛 / 合法性大损 / 派系动荡
- 变法 UI：在 M4 经济面板增加"变法进度"显示

> **注**：商鞅变法（前 359-338）与吴起变法（前 390-381）早于游戏起始年（前 260），已作为初始 traits 挂在 realm_qin/realm_chu。

### 9.3 验收标准

- 玩家在弱秦剧本里通过变法链能感受到"国家性格在变化"
- 失败的变法不是简单回滚，而是**留下伤痕**（如赵武灵王末年沙丘之变伏笔）
- AI 在拥有革新者 + Builder 君主时**会主动**触发变法

### 9.4 M4.1 不做

- 完整变法树（每个流派一条）→ 推到 v1.x
- 玩家自定义变法内容 → 永远不做
- 变法的"自下而上"路径（民众请变法）→ 推到 v2

## 10. M4.2 · 灾害·贸易·派系 ✅ 已交付

> M4 主题的最后收尾：让和平期不再"光涨数"，引入扰动与博弈。

### 10.1 目标

**让和平不等于安全：天灾、商旅、派系斗争都是策略层。**

### 10.2 必须完成

#### 10.2.1 灾害与丰歉

- 灾害类型 5 种：丰年（+30% 农业）/ 歉年（-25%）/ 大旱、大水（局部毁灭）/ 蝗灾（区域）/ 瘟疫（人口骤降+军队打击）
- 触发频率：每年小灾 1–2 次（局部），每 5–10 年大灾 1 次
- 玩家应对 4 种：开仓赈灾 / 减税 / 强征 / 不管 → 各有民望与稳定性后果
- 与 M5 太守属性耦合：高 `政` 太守提升赈灾效率

#### 10.2.2 贸易商路

- 邑↔邑贸易路线（v1：一对一，无路由几何）
- 跨势力贸易需对方允许（外交开关）
- 战争自动切断敌对国之间的贸易
- 商路收益：路线两端国库 + 文化扩散（弱）
- 三大潜在干路骨架：西域 / 巴蜀 / 海上（v1 仅西域+巴蜀实装；海上推到 M9）

#### 10.2.3 内部派系（最小可行）

- 派系类型 6 种字段实装：君党 / 公族 / 世族 / 军功派 / 改革派 / 守旧派 / 外姓客卿
- 派系平衡条（一维 -100~+100）取代 M4 v1 的简化稳定值
- 失衡触发：政变 / 分裂事件（与 M5 继承危机共用通道）
- 政令对派系影响（如"军功爵"→ 军功派 +、世族 −）

### 10.3 验收标准

- 玩家会说："那年大旱我没赈灾，结果楚国攻过来时我后方乱了"——多系统耦合的叙事
- 田氏代齐式的"派系吞并主家"事件可在 AI vs AI 自动局发生
- 100 tick perf 仍在预算内（灾害不能让性能崩溃）

### 10.4 M4.2 不做

- 完整商品种类（盐、铁、玉、丹砂等专营）→ 推到 v1.x
- 派系专门 UI 面板 → v1 仅显示数值条
- 灾害的"事件链"展开（如瘟疫 → 邻国封锁 → 战争）→ v1 仅单步影响
- 流民迁徙地理模拟 → 永远不做（按邑结算即可）

## 11. M6 · 文化与意识形态 ⏳ 待 M5

### 11.1 目标

**让软实力玩法成型：文化扩散、学宫、礼乐、威望。**

### 11.2 必须完成

#### 11.2.1 文化标签与认同

- 邑级 `cultural` 标签（chinese_qin / chinese_chu / chinese_qi / yi_dong / di_xirong …）
- 邑级 `cultural_identity_strength`（0–100，慢变）
- 文化转化机制：被征服 → 短期认同 ↓ → 中期可恢复 → 长期可转化（>50 年）
- 文化影响：低认同 → 起义概率 +、税收效率 −

#### 11.2.2 意识形态系统

- 6 种意识形态：法家 / 儒家 / 道家 / 墨家 / 纵横家 / 兵家
- 势力 `ideology_lean` 多维向量（每方向 0–100）
- 倾斜来源：① 君主性格（M5）② 主政人才流派（M5）③ 政令历史（M4 v1）④ 学宫派系（M6 自身）

#### 11.2.3 学宫系统

- 学宫建筑（首批 3 处历史名所：稷下学宫 / 西河学宫 / 兰陵学宫）
- 学宫每年产出人才（按势力意识形态倾向产出对应专长，**取代 M5 的剧本随机生成**）
- 学宫为意识形态输出基地：影响相邻势力学子流向

#### 11.2.4 礼乐与威望（Prestige）

- 势力 `prestige`（0–100）实装 — 这是 M4 v1 推迟的字段
- 来源：合法性、文化扩散、武功、礼制守护、举办盟会
- 用途：外交筹码、册封他国能力、文化吸引力

#### 11.2.5 朝贡与册封（接入 M3）

- M3 的 `tribute` 现在产生**文化拉力**（朝贡国对宗主认同 +）
- 周天子册封事件链 v1：册爵→受封→纳贡循环
- 僭号称王事件（魏惠王式）：威望惩罚 + 周王关系大跌

### 11.3 验收标准

- 玩家选鲁国，可走"文化胜利"路径（教化 ≥ 60% 邑认同 → 触发 §[`08-victory.md`](./08-victory.md) §3.4 路径）
- 战国客卿现象自然涌现：人才在不同学宫之间流动
- AI 之间会因为意识形态相近而结盟更顺畅

### 11.4 M6 不做

- 文化"语言"系统（春秋战国都用雅言，无需细分）
- 学宫的玩家直接控制学派内容 → v2
- 文化扩散的"地理模型"（按邻接图传播即可，不做 GIS 真实模拟）
- 受命于天 / 九鼎胜利路径完整事件链 → 推到 M9（与剧本内容打包）

## 12. M7 · 谍报系统 ⏳ 待 M5

> 谍报是中国大战略的"暗线"，但**不是 MVP 必需**。M7 仅做基础三件套。

### 12.1 目标

**让博弈有暗线：你不知道我盯着你，但你能怀疑。**

### 12.2 必须完成

- **情报覆盖度**：每对势力之间维护 0–100 的覆盖度（影响地图上对方信息可见性）
- **谍报动作 v1（仅 4 种）**：
  - 侦察（低风险，提升覆盖度）
  - 散布谣言（中风险，降稳定性）
  - 离间（中风险，降敌方人才忠诚）
  - 反谍（防御方动作，提升识破概率）
- **间者人才类型**：M5 已有 `间者` 专长字段；M7 让其实际工作
- **失败暴露机制**：失败的谍报任务**必须暴露**给被谋方，触发外交灾难

### 12.3 验收标准

- 玩家可观察"敌方间者活动"通知
- AI 在低信任时会主动散布谣言（性格 = Schemer 的 AI 倾向更强）
- 离间事件确实降低对方人才忠诚（与 M5 离心通道相通）

### 12.4 M7 不做（明确推迟到 v1.x）

- 策反郡守（高风险、玩法复杂）
- 策反将领（临阵倒戈）
- 暗杀人才（极高风险）
- 暗杀君主（败露 = 外交灾难，需要完整事件链）
- 窃取战法 / 文化（需要完整科技树）

> 注：MVP "Cut List"（§16.2）中"完整谍报系统"是优先砍的项目之一。M7 只做基础三件套并不影响 MVP 完整性。

## 15. 风险与权衡（Risks & Trade-offs）

### 15.1 主要风险

| 风险 | 影响 | 应对 |
|------|------|------|
| **M5 依赖根膨胀** | M5 一旦延期，M4.1/M6/M7 全停 | M5 严格按 §8.4 边界，决策点 D1-D3 落定后立刻动 |
| **AI 设计过于雄心** | 实现复杂、平衡困难 | M1–M5 用简易 AI，M8 集中精力扩 8 性格 |
| **历史考据成本超预期** | 内容拖延 | 先用"形似神在"原则放过细节，迭代式完善 |
| **网页性能瓶颈** | 影响 5x 速度体验 | M0 早测；必要时引入 Web Worker |
| **状态管理复杂度爆炸** | 难以调试 | 严格分层、Immer 不可变、单元测试覆盖关键路径 |
| **范围蔓延** | 永远做不完 | 严守里程碑，**砍**而不是**挪** |
| **单人开发疲劳** | 烧光热情 | 里程碑式发布，每个 M 都给玩家试玩 |

### 15.2 必要时砍什么（Cut List）

如果时间不够，按**优先级从低到高**砍：

1. ❌ 第二条胜利路径（仅保留"一统"）
2. ❌ M7 完整谍报系统（保留侦察+反谍，详见 §12.4）
3. ❌ M6 文化扩散（仅做基础认同度，跳过学宫产人才）
4. ❌ M4.1 完整变法事件链（仅做商鞅一例）
5. ❌ M6 周王册封事件链（保留 M3 已有的最简版本）
6. ❌ 联姻 / 质子（合并到 M3 已有外交基本动作）
7. ❌ M4.2 派系系统（保留稳定性数值即可）

> **绝不**砍：核心循环、地图、涂色、暂停、单一胜利路径、单一首发剧本、**M5 人才骨架**（这是 M4.1+ 的依赖根，砍了等于砍 M4.1/M6/M7）。

### 15.3 关键决策点（Go/No-Go）

每个里程碑结束应进行 **Go/No-Go 评审**：
- 当前手感是否符合愿景？
- 玩家测试反馈是否积极？
- 性能是否达标？
- 是否需要砍范围？

> 不达标就停下来调整，**绝不**带着已知问题进入下一里程碑。

## 16. MVP 范围定义（Minimum Viable Product）

### 16.1 必须有（Must Have）

- ✅ 完整可玩的春秋战国剧本（七雄 + 周）
- ✅ 核心循环（时间、暂停、涂色、行军） **[M0/M1 ✅]**
- ✅ 战争系统（兵种、关隘、将领、议和） **[M2 ✅]**
- ✅ 外交系统（盟约、宣战、议和） **[M3 ✅]**
- ✅ 经济系统（粮、钱、税） **[M4 v1 ✅]**
- ✅ 内政系统（政令、太守、人口） **[M4 v1 ✅；变法在 M4.1，灾害/派系在 M4.2]**
- ✅ 君主与人才（君主能力、关键人物、任免） **[M5 待做]**
- ✅ 一统胜利路径（其他路径 v1.x 逐步加入）
- ✅ AI（多层决策、性格驱动、不作弊） **[M3 已有 3 性格；M5/M8 扩到 8 性格]**
- ✅ 存档系统 **[M11]**
- ✅ 主屏 UI 完整 **[M10 持续]**
- ✅ 引导剧本

### 16.2 可以没有（Should Have，但非必须）

- 谍报全功能（保留基础侦察） **[M7 仅做 4 种基础动作]**
- 文化扩散 **[M6，但 §15.2 列为可砍]**
- 多胜利路径 **[首发仅"一统"和"守土"]**
- 史书百科完整内容

### 16.3 暂不做（Won't Have v1）

- 联机多人
- 多语言（英文）
- 移动端
- 模组工具
- 第二剧本（三国 / 五代十国）
- 战斗沙盘可视化
- DLC / 付费内容

## 17. 发布策略（Release Strategy）

### 17.1 Alpha（M5/M4.1/M4.2 已完成；Alpha 候选阶段）

- 内部团队 + 5-10 位铁杆历史游戏玩家
- 重点收集"手感"、"AI 行为"、"君主代际更替"反馈
- 关键验证：变法事件链能否让玩家产生"我重写了历史"的体验？

### 17.2 Beta（M9 完成时）

- 公开测试，邀请制 50–200 人
- 重点收集 Bug、平衡、内容反馈

### 17.3 1.0（M12 完成时）

- 公开发布
- 网页直接可玩，无需注册
- 收集长期反馈准备 v1.x

### 17.4 v1.x 与 v2

- v1.x：补充胜利路径、加深内容、修复反馈问题
- v2：第二剧本（三国或春秋早期）

## 18. 时间预估的诚实陈述

> **大战略游戏的开发时间被低估几乎是行业铁律**。本文给出的估算是**乐观情况**，实际可能延长 50%–100%。

不要为了赶进度牺牲核心体验。**晚一年但好玩**，远胜过**按时但平庸**。
