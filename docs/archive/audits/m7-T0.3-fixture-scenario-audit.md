## 2. M7 Fields to Add (Wave 1 T1.4)

M7 espionage system will add **3 new Maps** to World:

1. **intelligenceCoverage**: ReadonlyMap<CoverageKey, number>
   - Type: `CoverageKey = string` (format: `${observerRealmId}__${targetRealmId}`)
   - Values: 0-100 (coverage percentage)
   - Size: 8 realms × 7 targets = 56 directional entries (not symmetric)
   - Purpose: Track how much each realm knows about each other realm

2. **spyMissions**: ReadonlyMap<SpyMissionId, SpyMission>
   - Type: `SpyMissionId = string`
   - Values: SpyMission objects with id, spyId, targetRealmId, missionType, status, etc.
   - Purpose: Track active spy missions (infiltration, sabotage, assassination, etc.)

3. **counterIntelStates**: ReadonlyMap<RealmId, CounterIntelState>
   - Type: `RealmId` → CounterIntelState
   - Values: CounterIntelState with counterIntelBudget, detectionChance, etc.
   - Purpose: Track each realm's counter-intelligence capability and state

## 5. Factory Pattern Notes (for T1.4)

### createWorldFromM1Data Pattern (src/engine/world/factory.ts, lines 295-464)

The factory function demonstrates the pattern for initializing Maps from M1DataV6:

1. **Load data**: `M1DataSchemaV6.parse(data)` validates schema
2. **Build Maps**: For each collection in data, create a Map<Id, Entity>:
   ```typescript
   const generals = new Map<GeneralId, General>()
   for (const gen of data.generals as General[]) {
     generals.set(gen.id, gen)
   }
   ```
3. **Return World**: Assign all Maps to World object (lines 411-464)
